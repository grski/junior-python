## README File - What It Should Contain and How It Should Look

There's one thing that every proper project should contain. A README file. A well-written, readable, and comprehensive README file is the basic form of documentation for every project, containing elementary information about the project itself, but in my opinion, not only that. A well-written README file should contain a bit more. What exactly? I'll describe it now.

### Technology

Usually, when we work with README files, they have the `.md` extension, meaning according to convention, we write them in Markdown. Markdown is something that allows us to format text in a way. Using it in README is not a requirement, but it's a fairly common standard that makes our lives a bit easier and gives us more possibilities than a plain text file.
